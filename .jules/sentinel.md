## 2024-05-24 - Mass Assignment via Object.assign(req.body)
**Vulnerability:** A Mass Assignment vulnerability was identified in the `PUT /api/transactions/:id` route (`backend/src/routes/transactions.ts`). The endpoint used `Object.assign(transaction, req.body)` to update a database document.
**Learning:** Even when `express-validator` checks are present, the original `req.body` object still contains unvalidated properties sent by the client. Using `req.body` directly in an update operation allows attackers to modify fields they shouldn't have access to, such as internal identifiers (`userId`), critical security parameters (`isVerified`), or internal logic states. This breaks the security boundary of the API.
**Prevention:** Never use `Object.assign(doc, req.body)` or pass `req.body` directly to database update functions (like `Model.findByIdAndUpdate(id, req.body)`). Always explicitly assign allowed fields or use `matchedData(req, { locations: ['body'] })` to extract only the properties that have been validated by `express-validator`.
